Risks & mitigations

Every risk is architectural,
not operational.

Every failure mode has a corresponding architectural solution. Risk is not ignored — it is designed around.

DATA QUALITY Wrong data leads to bad decisions
Shadow Queries
5% of dispatches are known-answer tests. Sensors never know which. Uncertainty is the deterrent.
Accuracy Vectors
Per-sensor, per-query-type scores update after every outcome. Bad sensors filtered automatically.
AI Photo Check + Confidence Scoring
Claude reviews every submission. Results carry scored probability — 0.91, not just "verified". Low confidence triggers re-query.
LIABILITY Wrong output causes measurable harm
Observational Data Only
Workers collect publicly observable retail facts. Joshua Tandy, J.D. has confirmed this framework does not constitute HIPAA-regulated activity.
Cryptographic Provenance
SHA-256 hashed chain of custody on every query. What was observed, when, by whom — immutable record shifts liability appropriately.
SUPPLY QUALITY Reliability degrades as network grows
Tiered Access
High-accuracy sensors get priority dispatch and better pay. The system self-selects for quality at scale.
GPS Anti-Spoofing + Worker Isolation
Velocity detection catches location fraud. Sensors never know which queries are shadow tests — gaming the system is structurally irrational.
REGULATORY Worker classification or privacy challenges
Independent Contractor Model
Sensors are independent contractors completing discrete observation tasks — same structure as gig platforms operating at global scale.
Entity Separation + Platform ToS
Proxy operates as a legally distinct entity. Standard SaaS ToS covering worker data, query outputs, and data retention — drafted before first real query dispatched.

Common questions

What people ask
before they sign up.

Is this just gig work with a fancier name?
No. Gig platforms dispatch humans to complete tasks. Proxy dispatches human sensors to return structured, confidence-scored, provenance-verified data that AI agents consume directly. Every observation is stored permanently as a Truth Atom in the Ground Truth Database. The sensor is not the product — the data layer is.
Can an AI agent use Proxy without a human operator in the loop?
Yes. Agents can query, set the price, make the payment, and manage physical-world verifications autonomously via REST API or MCP server — no human required. The Agent Operator role exists for teams who want oversight, but it is not required.
How do you know the sensor data is accurate?
Every result carries a confidence score combining the sensor's accuracy history, AI-assisted proof review, and location verification. Results include a cryptographic provenance record — a chain of custody from physical observation to structured output. Shadow queries (known-answer tests dispatched anonymously) continuously calibrate sensor accuracy without their knowledge.
What happens when truth goes stale?
Every Truth Atom has a calibrated half-life based on query type — drug availability decays faster than permit status, for example. Proxy tracks decay in real time and alerts Agent Operators before confidence drops below action threshold. You never act on stale data without knowing it.
Where is Proxy available and what's the rollout plan?
Salt Lake City pilot is live with a registered sensor network. Santiago, Chile is next in partnership with an established technology operator. Additional markets open after each pilot is proven. We expand depth before breadth — one working city beats ten half-working ones.
What query types does Proxy support?
Any physical-world observation that requires a human to be present at a specific location at a known time. Current and planned query types span healthcare (drug availability, formulary checks, infusion center capacity, storage conditions), information markets (construction milestones, product launch confirmation, event attendance), retail (shelf inventory, pricing, promotional compliance), compliance and audit (site readiness, physical controls, chain of custody), real estate and infrastructure (construction progress, occupancy, asset condition), and logistics (delivery confirmation with visual proof, cold chain verification). The common thread: no database, API, or model can substitute for a verified human physically present at a location with provenance. That's the gap Proxy closes.
Won't robots and cameras eventually replace this?
No — and the reason is structural, not temporary. A camera network or robot fleet requires capital, installation time, permitting, maintenance, and geographic pre-commitment. Proxy sensors are people who already live in every city on earth. Geographic coverage is instantaneous. A robot cannot be dispatched to Salt Lake City tomorrow. A Proxy sensor can be dispatched in 60 seconds. Beyond deployment speed, physical verification often requires judgment — a label that's partially obscured, a situation that requires asking a staff member, an edge case no sensor was pre-configured to handle. And crucially: the data moat is sensor-agnostic. Even if IoT density increases, the Ground Truth Database compounds. The moat is the data, not the human specifically.
